### 飞书文档API教训总结
1. `create_table_with_values` 的 `after_block_id` 不可靠，新表格可能跑到文档末尾
2. `write` 全量覆盖**无法清除已有的原生 table blocks**（block_type=31）
3. `delete_block` 对 table 子blocks 返回 400/404
4. **唯一可靠方案：创建全新文档，从头按顺序写入**
5. 每个表格最大约6行（超过报400），需拆分
6. 9列可以成功写入
